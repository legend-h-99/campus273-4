قاعدة البيانات — db push + seed

إنشاء الجداول وتعبئة البيانات التجريبية

🖥 على السيرفر ◀
مهم: لازم تتأكد إن db push يقول "Your database is now in sync" وليس "No pending migrations". لو قال migrations يعني الصورة القديمة لسّا مُستخدمة.

5.1 — أوقف كل شيء واحذف البيانات القديمة:

cd /opt/campus27
docker compose -f docker-compose.prod.yml --env-file .env.production down -v

5.2 — احذف الصور القديمة وأعد البناء:

docker rmi $(docker images -q 'campus27*') 2>/dev/null; true
docker compose -f docker-compose.prod.yml --env-file .env.production build --no-cache
⏳ ياخذ ~3-5 دقائق للبناء

5.3 — شغّل قاعدة البيانات:

docker compose -f docker-compose.prod.yml --env-file .env.production up -d db
sleep 15

5.4 — أنشئ الجداول (db push):

docker compose -f docker-compose.prod.yml --env-file .env.production --profile migrate run --rm migrator
✓ Your database is now in sync with your Prisma schema.

5.5 — تعبئة البيانات (Seed):

docker compose -f docker-compose.prod.yml --env-file .env.production --profile migrate run --rm migrator npx tsx prisma/seed.ts
✓ Seed completed successfully!

5.6 — تحقق من المستخدمين:

docker compose -f docker-compose.prod.yml --env-file .env.production exec db \
  psql -U campus27 -d campus27 -c "SELECT email, role FROM users LIMIT 5;"
admin@campus27.sa | SUPER_ADMIN
dean@campus27.sa | DEAN
...

5.7 — شغّل كل الخدمات:

docker compose -f docker-compose.prod.yml --env-file .env.production up -d

6.3 — تحديث NEXTAUTH_URL (على السيرفر):

6.4 — Nginx config (على السيرفر):

cat > /opt/campus27/deploy/nginx/conf.d/cranl.conf << 'EOF'
# cranl.com redirect to app.cranl.com
server {
  listen 80;
  server_name cranl.com www.cranl.com;
  return 301 https://app.cranl.com$request_uri;
}

# app.cranl.com main app
server {
  listen 80;
  server_name app.cranl.com;
  location / {
    proxy_pass http://app:3000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
  location /api/ {
    limit_req zone=api burst=20 nodelay;
    proxy_pass http://app:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
  location /api/auth/ {
    limit_req zone=login burst=5 nodelay;
    proxy_pass http://app:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
  location /_next/static/ {
    proxy_pass http://app:3000;
    add_header Cache-Control "public, max-age=31536000, immutable";
  }
}
EOF

6.5 — أعد تشغيل الخدمات:

cd /opt/campus27
docker compose -f docker-compose.prod.yml --env-file .env.production restart nginx app

7.2 — تسجيل الدخول:

بيانات الدخول

🌐 https://app.cranl.com
Email: admin@campus27.sa

7.3 — تسجيل دخول من سطر الأوامر:

# 1. جيب CSRF token
CSRF=$(curl -s -c /tmp/c.txt http://localhost:3000/api/auth/csrf | grep -o '"csrfToken":"[^"]*"' | cut -d'"' -f4)

# 2. سجّل دخول
curl -s -c /tmp/c.txt -b /tmp/c.txt \
  -X POST http://localhost:3000/api/auth/callback/credentials \
  -d "csrfToken=${CSRF}&email=admin@campus27.sa&password=123456" \
  -w "\nHTTP: %{http_code}\n" -o /dev/null

# 3. تحقق من الجلسة
curl -s -b /tmp/c.txt http://localhost:3000/api/auth/session
{"user":{"id":"...","email":"admin@campus27.sa","role":"SUPER_ADMIN",...}}
